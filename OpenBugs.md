## Tiberian Sun
* **BUG-011:** ReCnC Windows packaging did not ship a **Tiberian Sun launcher** (`TiberianSun.exe`) or Start Menu / Desktop shortcuts for TS, unlike Red Alert, Tiberian Dawn, and Dune 2000 — so common mod/hotkey docs that assume a per-mod entry point were hard to use. **Mitigation:** `build-launchers.ps1` now builds `TiberianSun.exe` (mod `ts`), bundles `mods/ts` + `mods/ts-content`, and the NSIS installer registers shortcuts and `openra-ts-${TAG}` URI handling. (TS mod hotkeys live under `mods/ts/hotkeys.yaml`; they load with the mod once launched.)
* **Crash (resolved 2026-04-26):** `exception-2026-04-26T020823Z.log` — `DivideByZeroException` in `BodyOrientationInfo.QuantizeFacing` during render (`WithIdleOverlay` → `WithShadow`) on TS map *Tiers of Sorrow*. **Cause:** TS `^VoxelActor` / `^Aircraft` use `BodyOrientation.QuantizedFacings: 0` (disable facing quantization); `QuantizeOrientation` respected `0` but `QuantizeFacing` still divided by zero. **Fix:** early return in [BodyOrientation.cs](OpenRA-source/OpenRA.Mods.Common/Traits/BodyOrientation.cs) when `facings == 0`. Rebuild engine and re-test; other crashes: `%AppData%\OpenRA\Logs`.
* **Shadows:** Infantry (possibly other voxel or sprite units) cast shadows at angles that do not match building shadows — looks inconsistent; may be lighting / voxel floor vs structure rendering.
* **BUG-008 (TS) / BUG-013 — TS harvester, refinery, docking (merged):** Multiple harvesters pile onto a single refinery instead of spreading or preferring their **home** refinery; harvester **queue** / docking oddities under **Improved** docking selector (`DockingStrategyManager` / lobby algorithm option). **Design question:** split docking logic **harvester vs heli/plane vs service depot** (reporter: **service depots behave fine**) vs one shared path? Expected: sensible assignment + pathfinding; overlaps OpenRA harvester–refinery code and TS `shared-structures.yaml` / `shared-vehicles.yaml` **HARV**/**PROC**.
* **BUG-012 / Titan (MMCH) weapon barrel:** Voxel **barrel is not shown correctly** as the unit **rolls out of the vehicle factory**; the same wrong barrel angle appears when the unit is **moving normally** at the **same facing** (not only during exit). Titan is **`MMCH`** with `WithSpriteTurret` + `WithVoxelBarrel` in [mods/ts/rules/gdi-vehicles.yaml](OpenRA-source/mods/ts/rules/gdi-vehicles.yaml) (build queue: **vehicle factory** / `gaweap` family — not refinery).
